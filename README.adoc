= AAP with EDB Postgres -- Multi-Datacenter DR Demo
:toc:
:toc-placement!:

Showroom demo lab for **Ansible Automation Platform 2.6** with **EDB Postgres Advanced Server 16** in an active/passive multi-datacenter disaster recovery architecture.

toc::[]

== Purpose

* Demonstrate **active/passive AAP** with **EDB Postgres streaming replication** and **automated failover** to customers
* Show **RTO under 5 minutes** and **RPO under 5 seconds** in a live demo
* Run on **5 RHEL VMs** representing two logical datacenters

== Lab topology

[cols="1,2,2"]
|===
|VM |Role |Datacenter

|`control`
|AAP Controller (active)
|DC1

|`pg-dc1`
|EDB Postgres primary
|DC1

|`pg-dc2`
|EDB Postgres standby (streaming replica)
|DC2

|`aap-dc2`
|AAP Controller (standby, services stopped)
|DC2

|`vscode`
|VS Code IDE
|--
|===

== Build the static site locally

Requires `podman` (or adjust `utilities/lab-build` for Docker).

[source,bash]
----
./utilities/lab-build
./utilities/lab-serve
# Open http://localhost:8080
----

== Getting started

. Review `config/REQUIRED_INPUTS.txt` for environment variables and prerequisites
. Ensure EDB Postgres Advanced Server repo access is configured
. Run `setup-automation/main.yml` to provision VMs
. Access the Showroom lab and work through modules 1-5
